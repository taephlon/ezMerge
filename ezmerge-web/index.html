Welcome to ezMerge

Gentoo Linux is renowned for its incredible flexibility and source-based optimization. However, managing third-party package repositories (known as overlays) is historically one of the most error-prone tasks for both beginners and seasoned administrators.

The ezMerge ecosystem bridges this gap by creating helper utilities and standard guidelines to make overlay package installation feel identical to pulling from Gentoo's official package repository.

Core Architecture

  • ezmerge-cli: A terminal-based assistant that automates overlay discovery, sync, keyword acceptances, and USE flag resolutions.
  • ezmerge-overlay: Our own curated repository hosting stable ebuilds with automated quality checks.
  • ezmerge-web: This package index portal, serving package details, metadata, and copyable install instructions.

Installing ezmerge-cli

ezMerge CLI is packaged inside our curated overlay, as well as distributed as a standalone Rust binary.

Method 1: From the ezmerge-overlay (Recommended)

# Enable the ezmerge-overlay via eselect
eselect repository enable ezmerge-overlay
emaint sync -r ezmerge-overlay

# Emerge the package assistant
emerge --ask app-admin/ezmerge-cli

Method 2: Cargo install (For developers)

If you have Rust and Cargo configured on your Gentoo box, you can install the CLI tool directly from source:

git clone https://github.com/ezmerge/ezMerge.git
cd ezMerge/ezmerge-cli
cargo install --path .

How ezMerge Interacts with Portage

ezMerge is designed as a companion, not a replacement. It respects the standard file layouts of Gentoo Portage and leaves your configuration in a completely valid, standard state.

Configuration Files Modified

When you confirm an install with custom configurations, ezmerge-cli writes to specific, isolated config slices inside /etc/portage/:

  • /etc/portage/package.use/ezmerge: Custom USE flag entries.
  • /etc/portage/package.accept_keywords/ezmerge: Unmasked testing keywords (e.g. ~amd64).

This layout ensures that you can undo or review all changes made by ezMerge simply by deleting these files or using ezmerge undo.

Overlay Quality & Trust Scores

To prevent users from breaking their systems with untrustworthy or broken overlays, ezMerge queries a curated database to compute a Trust Score (out of 5.0) for active overlays.

Trust Score Criteria

Signal Weight Description
Recent Activity 30% Overlays updated in the last 7 days receive high scores. Dead overlays are penalized.
Maintainer count 35% Collaborative repositories (like GURU) have multiple reviewers, lowering security and packaging risks.
CI Build Status 35% Overlays that run standard Gentoo validation (pkgcheck) and pass compilation tests.
